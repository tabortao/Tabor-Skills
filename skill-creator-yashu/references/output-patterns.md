# 输出模式

使用这些模式当 skills 需要产生一致、高质量的输出。

## 模板模式

为输出格式提供模板。根据需要将严格程度与需求匹配。

### 对于严格要求（如 API 响应或数据格式）

```markdown
## 报告结构

始终使用此确切模板结构：

# [分析标题]

## 执行摘要
[关键发现的一段概述]

## 关键发现
- 带有支持数据的发现 1
- 带有支持数据的发现 2
- 带有支持数据的发现 3

## 建议
1. 具体的可操作建议
2. 具体的可操作建议
```

### 对于灵活指导（当适应性有用时）

```markdown
## 报告结构

这是一个合理的默认格式，但请运用你的最佳判断：

# [分析标题]

## 执行摘要
[概述]

## 关键发现
[根据你发现的内容调整部分]

## 建议
[针对具体情境定制]

根据分析类型调整部分。
```

## 示例模式

对于输出质量依赖示例的 skill，提供输入/输出对：

```markdown
## 提交信息格式

按照以下示例生成提交信息：

**示例 1：**
输入：添加了使用 JWT 令牌的用户认证
输出：
```
feat(auth): implement JWT-based authentication

Add login endpoint and token validation middleware
```

**示例 2：**
输入：修复了报告中日期显示不正确的 bug
输出：
```
fix(reports): correct date formatting in timezone conversion

Use UTC timestamps consistently across report generation
```

遵循此风格：type(scope): 简短描述，然后是详细解释。
```

示例比单独的描述更能帮助 AI 理解期望的风格和详细程度。

## 检查清单模式

对于需要验证的任务：

```markdown
## 代码审查检查清单

在批准代码之前，验证以下各项：

- [ ] 代码遵循项目编码规范
- [ ] 所有函数都有文档字符串
- [ ] 错误处理已适当实现
- [ ] 单元测试已添加/更新
- [ ] 没有安全漏洞
- [ ] 性能影响已考虑
```

## 表格模式

对于结构化数据输出：

```markdown
## 输出格式

以表格形式呈现结果：

| 字段 | 值 | 说明 |
|------|-----|------|
| 名称 | [值] | [说明] |
| 类型 | [值] | [说明] |
| 状态 | [值] | [说明] |
```
