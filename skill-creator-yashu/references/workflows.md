# 工作流程模式

## 顺序工作流程

对于复杂任务，将操作分解为清晰的顺序步骤。在 SKILL.md 开头给出流程概览通常很有帮助：

```markdown
填写 PDF 表单涉及以下步骤：

1. 分析表单（运行 analyze_form.py）
2. 创建字段映射（编辑 fields.json）
3. 验证映射（运行 validate_fields.py）
4. 填写表单（运行 fill_form.py）
5. 验证输出（运行 verify_output.py）
```

## 条件工作流程

对于有分支逻辑的任务，指导 AI 通过决策点：

```markdown
1. 确定修改类型：
   **创建新内容？** → 遵循下面的"创建工作流程"
   **编辑现有内容？** → 遵循下面的"编辑工作流程"

2. 创建工作流程：[步骤]
3. 编辑工作流程：[步骤]
```

## 决策树模式

对于有多种路径的复杂任务：

```markdown
## 工作流程决策树

**用户需要创建新文档？**
→ 遵循 [创建文档](#创建文档) 流程

**用户需要编辑现有文档？**
→ 遵循 [编辑文档](#编辑文档) 流程

**用户需要分析文档内容？**
→ 遵循 [分析文档](#分析文档) 流程
```

## 循环工作流程

对于需要重复执行的任务：

```markdown
## 批量处理工作流程

对于每个文件：

1. 读取文件内容
2. 应用转换
3. 验证输出
4. 保存结果

重复直到所有文件处理完成。
```

## 错误处理

工作流程执行失败时的处理方式：

### 步骤失败处理

当某个步骤执行失败时：

1. **记录错误信息** - 保存错误日志和上下文
2. **决定处理方式**：
   - **可恢复错误** → 重试该步骤（最多 3 次）
   - **不可恢复错误** → 终止流程并报告
3. **清理临时资源** - 删除或回滚已创建的中间文件

### 常见错误场景

| 错误类型     | 处理方式                       |
| ------------ | ------------------------------ |
| 文件不存在   | 检查路径是否正确，确认文件权限 |
| 脚本执行失败 | 查看错误输出，修复后重试       |
| 依赖缺失     | 安装所需依赖后重试             |
| 超时         | 增加超时时间或优化处理逻辑     |
