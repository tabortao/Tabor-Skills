# 检查规范

本文档供 AI 分析时参考，说明各项检查标准。

## 1. Frontmatter 格式

### 必需字段

| 字段        | 要求                                    |
| ----------- | --------------------------------------- |
| name        | 小写字母/数字/连字符，1-64 字符         |
| description | 1-1024 字符，必须包含功能描述和触发条件 |

### description 格式

必须包含两部分，用 `何时使用：` 分隔：

- 功能描述 - 这个技能是做什么的
- 何时使用 - 用户说什么话时触发

**正确示例**：

```yaml
description: 分析 Skill 文档质量问题。何时使用：当用户说"检查 skill 质量"时。
```

### metadata 字段

| 字段    | 格式                |
| ------- | ------------------- |
| author  | 字符串              |
| updated | YYYY-MM-DD HH:MM:SS |
| version | 语义化版本号        |

## 2. 渐进式披露结构

| 层级                    | 建议大小    |
| ----------------------- | ----------- |
| Metadata                | ~100 tokens |
| Instructions (SKILL.md) | < 500 行    |
| Resources               | 按需加载    |

## 3. 文件引用完整性

检查 SKILL.md 中引用的文件是否存在：

- `scripts/` 下的脚本文件
- `references/` 下的参考文档
- `assets/` 下的资源文件

## 4. AI 友好性检查

| 检查项                 | 重要性 |
| ---------------------- | ------ |
| 明确的指令（祈使句）   | ⭐⭐⭐ |
| 具体的示例             | ⭐⭐⭐ |
| 决策逻辑               | ⭐⭐   |
| 输出格式说明           | ⭐⭐   |
| 错误处理说明           | ⭐⭐   |
| 避免长段落（>500字符） | ⭐     |

## 5. 使用说明完整性

必须包含使用说明章节，如：

- `## 如何使用这个 Skill`
- `## 使用说明`
- `## 触发条件与执行`

## 6. Token 效率

- 使用表格组织信息
- 使用列表代替长段落
- 避免重复内容
